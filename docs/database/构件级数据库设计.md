# 构件级数据库设计文档

## 概述

本文档描述构件级建筑数据库的设计方案，采用多层次架构实现建筑的模块化生成。

---

## 架构设计

### 多层次构件架构

```
层次4: Complex (建筑群)    → 村庄、基地、城镇
层次3: Building (完整建筑) → 住宅、塔楼、商店、神庙
层次2: Composite (复合构件) → 房间、楼层、走廊
层次1: Atomic (原子构件)   → 屋顶、墙壁、装饰、基础、框架
```

### 设计理念

1. **复用性**: 原子构件可在多个建筑间复用
2. **可组合性**: 复合构件由原子构件组合而成
3. **可扩展性**: 易于添加新构件和风格
4. **向量检索**: 支持语义相似度检索

## 数据库表设计

### 核心表

#### 1. building_index (建筑索引表)

用于向量检索的索引表：

```sql
CREATE TABLE building_index (
    id TEXT PRIMARY KEY,              -- 建筑ID
    name TEXT,                        -- 建筑名称
    source_id TEXT,                   -- 来源原始数据ID
    
    -- 检索向量
    vector TEXT,                      -- 向量数据 (JSON数组, 384维)
    searchable_text TEXT,             -- 检索文本
    
    -- 快速筛选
    complexity_level TEXT,            -- atomic/composite/building/complex
    building_type TEXT,               -- house/tower/shop/temple
    style TEXT,                       -- asian/medieval/fantasy
    size_category TEXT,               -- small/medium/large
    width_range TEXT,                 -- [min, max]
    height_range TEXT,                -- [min, max]
    
    -- 摘要
    summary TEXT                      -- 一句话描述
);
```

**记录数**: 28条

#### 2. buildings (建筑实体表)

完整建筑的定义：

```sql
CREATE TABLE buildings (
    id TEXT PRIMARY KEY,
    building_type TEXT,               -- house/tower/shop/temple
    structure_type TEXT,              -- single_story/multi_story/tower
    
    -- 尺寸
    width INTEGER,
    height INTEGER,
    stories INTEGER,
    
    -- 风格
    style_tags TEXT,                  -- JSON数组: ["asian", "fantasy"]
    
    -- 结构组成
    structure TEXT,                   -- JSON: {foundation, stories, roof}
    
    -- 构件引用
    components TEXT                   -- JSON数组: [{ref, type}]
);
```

**记录数**: 28条

#### 3. atomic_components (原子构件表)

最小可复用单元：

```sql
CREATE TABLE atomic_components (
    id TEXT PRIMARY KEY,
    type TEXT,                        -- roof/wall/floor/decor/foundation/frame
    subtype TEXT,                     -- pagoda/gable/flat (屋顶子类型)
    
    -- 来源
    source_building TEXT,             -- 来源建筑ID
    
    -- 尺寸
    width INTEGER,
    height INTEGER,
    tile_count INTEGER,
    
    -- 材料
    tiles TEXT,                       -- JSON数组: 使用的主要方块
    walls TEXT,                       -- JSON数组: 使用的主要墙壁
    
    -- 生成规则
    pattern TEXT,                     -- solid/checkered/bordered
    placement_rule TEXT,              -- 放置规则描述
    
    -- 向量
    vector TEXT                       -- 构件向量 (可选)
);
```

**记录数**: 125条

#### 4. style_materials (风格材料映射表)

不同风格的推荐材料：

```sql
CREATE TABLE style_materials (
    style TEXT PRIMARY KEY,
    
    -- 方块推荐
    primary_tiles TEXT,               -- JSON数组
    secondary_tiles TEXT,             -- JSON数组
    accent_tiles TEXT,                -- JSON数组
    
    -- 墙壁推荐
    primary_walls TEXT,               -- JSON数组
    secondary_walls TEXT,             -- JSON数组
    
    -- 油漆方案
    paint_scheme TEXT,                -- JSON对象
    
    -- 特征元素
    characteristic_elements TEXT     -- JSON数组: ["lantern", "banner"]
);
```

**记录数**: 4条 (asian, medieval, fantasy, natural)

#### 5. vectors (统一向量存储表)

所有实体的向量统一存储：

```sql
CREATE TABLE vectors (
    id TEXT PRIMARY KEY,
    entity_type TEXT,                 -- building/component/style
    entity_level TEXT,                -- 1/2/3/4 对应构件层次
    vector TEXT,                      -- JSON数组 (384维)
    vector_model TEXT,                -- 生成模型: smart_keyword
    vector_dimension INTEGER,         -- 384
    searchable_text TEXT              -- 向量来源文本
);
```

**记录数**: 28条 (建筑向量)

## 向量系统

### 向量文件

```
Data/vectors/building_embeddings.json
```

格式：
```json
{
  "version": "3.0",
  "method": "building_semantic_v3",
  "dimension": 384,
  "generated": "2026-06-04T...",
  "count": 28,
  "embeddings": {
    "building_id_1": [0.123, -0.456, ...],
    "building_id_2": [0.234, -0.567, ...]
  }
}
```

### 向量生成方法

使用TF-IDF + 语义关键词增强：

```python
# 建筑类型关键词
BUILDING_TYPE_KEYWORDS = {
    "住宅": "house home residential living dwelling",
    "神庙": "temple shrine religious sacred worship",
    "商店": "shop store merchant trading market",
    "塔楼": "tower watchtower lookout vertical tall",
}

# 风格关键词
STYLE_KEYWORDS = {
    "中式": "chinese asian oriental dynasty pagoda",
    "日式": "japanese asian oriental sakura cherry",
    "奇幻": "fantasy magical enchanted mystical",
    "中世纪": "medieval castle knight stone brick",
}

# 尺寸关键词
SIZE_KEYWORDS = {
    "小型": "small tiny compact mini",
    "中型": "medium moderate average",
    "大型": "large big huge massive",
}
```

### 相似度测试结果

```
查询: '中式住宅 小型' → 匹配度 0.636 ✓
查询: '大型神庙 塔楼' → 匹配度 0.405 ✓
查询: '奇幻风格 建筑' → 匹配度 0.502 ✓
查询: '日式风格 小型住宅' → 匹配度 0.511 ✓✓
```

## C# 集成

### 初始化

```csharp
// 初始化知识库
KnowledgeBaseManager.Instance.Initialize();

// 加载向量
var kb = KnowledgeBaseManager.Instance.Components;
// 自动加载 Data/vectors/building_embeddings.json
```

### 检索建筑

```csharp
// 向量检索
var results = kb.SearchBuildingsByVector(queryVector, topK: 5);

// 条件筛选
var criteria = new BuildingSearchCriteria {
    style = "asian",
    building_type = "house",
    min_width = 15,
    max_width = 25
};
var buildings = kb.SearchBuildings(criteria);
```

### 获取构件

```csharp
// 获取建筑详情
var building = kb.GetBuildingDetail("building_id_1");

// 获取构件规则
var components = kb.GetComponents("building_id_1");

// 获取风格材料
var materials = kb.GetStyleMaterials("asian");
```
