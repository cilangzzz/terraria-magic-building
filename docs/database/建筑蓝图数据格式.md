# 建筑蓝图数据格式文档

## 概述

建筑蓝图（Building Schematic）是存储建筑区域数据的核心结构，基于TEdit导出的 `.TEditSch.json` 格式设计。Agent通过蓝图ID检索数据，理解建筑结构。

---

## 1. 数据表结构

### 1.1 building_schematics（蓝图主表）

存储蓝图元数据：

| 字段 | 类型 | 说明 | Agent用途 |
|------|------|------|-----------|
| id | INTEGER | 主键，蓝图ID | **检索键** |
| name | TEXT | 蓝图名称 | 标识 |
| display_name | TEXT | 显示名称 | 用户展示 |
| width | INTEGER | 宽度（格） | 尺寸判断 |
| height | INTEGER | 高度（格） | 尺寸判断 |
| category | TEXT | 分类(house/castle/tower等) | 分类检索 |
| style | TEXT | 风格(medieval/modern等) | 风格匹配 |
| biome_match | TEXT | 匹配生物群落 | 环境适配 |
| unique_tiles | INTEGER | 不同方块种类数 | 复杂度评估 |
| has_wires | INTEGER | 是否有电线 | 功能判断 |
| has_door | INTEGER | 是否有门 | 功能判断 |

### 1.2 schematic_tiles（方块数据表）

存储蓝图内每个格子的详细数据：

| 字段 | 类型 | 说明 |
|------|------|------|
| schematic_id | INTEGER | 关联蓝图ID |
| x | INTEGER | 相对X坐标 |
| y | INTEGER | 相对Y坐标 |
| tile_type | INTEGER | Tile ID（方块类型） |
| tile_u/tile_v | INTEGER | 帧坐标（多帧方块） |
| tile_color | INTEGER | 方块油漆颜色 |
| wall_type | INTEGER | Wall ID（墙壁类型） |
| wall_color | INTEGER | 墙壁油漆颜色 |
| liquid_type | INTEGER | 液体类型(0无/1水/2蜂蜜/3岩浆) |
| wire_red/blue/green/yellow | INTEGER | 电线状态 |
| actuator | INTEGER | 是否有制动器 |

### 1.3 schematic_analysis（分析表）

AI分析结果，用于语义检索：

| 字段 | 类型 | 说明 |
|------|------|------|
| schematic_id | INTEGER | 关联蓝图ID |
| primary_tiles | TEXT(JSON) | 主要方块统计 |
| primary_walls | TEXT(JSON) | 主要墙壁统计 |
| detected_style | TEXT | 检测到的风格 |
| semantic_text | TEXT | 语义描述文本 |

## 2. 蓝图ID检索流程

Agent通过蓝图ID获取建筑数据：

```
用户请求: "生成ID为1的建筑蓝图"

Agent工具调用:
get_schematic_by_id(id=1)
    → SQL查询: SELECT * FROM building_schematics WHERE id = 1
    → 获取元数据: name="1", width=16, height=10

get_schematic_tiles(schematic_id=1)
    → SQL查询: SELECT * FROM schematic_tiles WHERE schematic_id = 1
    → 获取方块列表: [(x=0,y=0,tile_type=187), ...]

构建建筑:
    for tile in tiles:
        WorldGen.PlaceTile(x + baseX, y + baseY, tile.tile_type)
```

## 3. 数据格式详解

### 3.1 原始JSON格式（TEditSch.json）

```json
{
  "name": "1",
  "width": 16,
  "height": 10,
  "tiles": [
    [  // 第一行 (y=0)
      {
        "active": true,
        "type": 187,          // Tile ID
        "u": 2682,            // 帧坐标X
        "v": 0,               // 帧坐标Y
        "wall": null,
        "tile_color": null,
        "wires": {"red": false, "blue": false, "green": false, "yellow": false}
      },
      // ... 16个格子
    ],
    // ... 10行
  ]
}
```

### 3.2 数据库存储格式

```sql
-- 元数据
INSERT INTO building_schematics (id, name, width, height) VALUES (1, '1', 16, 10);

-- 方块数据（压缩存储）
INSERT INTO schematic_tiles (schematic_id, x, y, tile_type, wall_type) VALUES
(1, 0, 0, 187, NULL),
(1, 1, 0, 187, NULL),
(1, 2, 0, NULL, 2),   -- 空方块，有墙壁
...
```

## 4. Agent工具定义

### 4.1 get_schematic_by_id

按ID获取蓝图元数据：

```csharp
private ToolResult GetSchematicById(int id, KnowledgeBaseManager kb)
{
    var schematic = kb.Schematics.GetById(id);
    return new ToolResult { Content = Serialize(schematic) };
}
```

返回示例：
```json
{
  "id": 1,
  "name": "1",
  "width": 16,
  "height": 10,
  "unique_tiles": 8,
  "primary_tiles": [{"id": 0, "count": 17, "name": "Dirt"}],
  "style": "natural"
}
```

### 4.2 search_schematics

语义搜索蓝图：

```csharp
private ToolResult SearchSchematics(string style, string category, int maxSize, KnowledgeBaseManager kb)
{
    // Step1: SQL过滤
    var candidates = kb.Schematics.Search(category, maxSize);

    // Step2: 向量匹配
    if (!string.IsNullOrEmpty(style) && kb.Vectors.IsInitialized)
    {
        candidates = kb.Vectors.SearchSchematicsSemantic(candidates, style, 10);
    }

    return new ToolResult { Content = Serialize(candidates) };
}
```

### 4.3 get_schematic_tiles

获取蓝图方块数据用于放置：

```csharp
private ToolResult GetSchematicTiles(int schematicId, KnowledgeBaseManager kb)
{
    var tiles = kb.Schematics.GetTiles(schematicId);
    // 返回方块列表，用于WorldGen放置
    return new ToolResult { Content = Serialize(tiles) };
}
```

## 5. 向量检索

### 5.1 向量生成

蓝图向量基于以下特征生成：
- 尺寸 (width x height)
- 主要方块类型和数量
- 风格标签
- 分类标签

```python
# 语义文本示例
semantic_text = "尺寸16x10 泥土17个 natural 草地7个 wood 木材3个 泥墙11个"
embedding = vectorizer.vectorize(semantic_text)
```

### 5.2 检索示例

```
AI请求: "生成一个小型木屋"

Agent工具调用:
search_schematics(style="natural", category="house", maxSize=20)
    → SQL: category='house' AND width <= 20 AND height <= 20
    → 向量: style='natural' 匹配
    → 返回: ID=5 (小型木屋, 10x8)

get_schematic_tiles(schematic_id=5)
    → 返回方块列表

执行放置:
    for tile in tiles:
        WorldGen.PlaceTile(baseX + tile.x, baseY + tile.y, tile.tile_type)
```

## 7. 使用流程

### 7.1 导入蓝图

```bash
# 1. 将TEditSch.json放入指定目录
# 2. 运行导入脚本
cd Tools/database
python init_schematic_tables.py

# 或批量导入
python import_schematics.py --dir /path/to/schematics
```

### 7.2 生成向量

```bash
cd Tools/vector
python generate_schematic_embeddings.py
```

### 7.3 Agent调用

```
用户: "在这里生成一个中世纪风格的小房子"

Agent:
1. search_schematics(style="medieval", category="house", maxSize=15)
   → 返回匹配的蓝图ID列表

2. get_schematic_by_id(id=选中的ID)
   → 获取蓝图详细信息

3. get_schematic_tiles(schematic_id=选中的ID)
   → 获取方块数据

4. 执行WorldGen放置
```
