# 数据检索系统文档

## 概述

本模组实现了混合检索架构，结合SQL精确查询与向量语义匹配，为Agent工具提供智能数据检索能力。

**注意**: 项目已升级为构件级架构，向量系统现在使用统一的 `building_embeddings.json` 文件。

### 3.2 ComponentKnowledgeBase

构件级检索核心：

```csharp
public class ComponentKnowledgeBase
{
    // 向量检索
    public List<BuildingInfo> SearchBuildingsByVector(float[] queryVector, int topK)
    {
        return _buildingIndex
            .Select(b => (building: b, score: CosineSimilarity(b.vector, queryVector)))
            .OrderByDescending(s => s.score)
            .Take(topK)
            .Select(s => s.building)
            .ToList();
    }
    
    // 条件筛选
    public List<BuildingInfo> SearchBuildings(BuildingSearchCriteria criteria)
    {
        return _buildings.Where(b =>
            (criteria.style == null || b.style == criteria.style) &&
            (criteria.building_type == null || b.building_type == criteria.building_type) &&
            b.width >= criteria.min_width && b.width <= criteria.max_width
        ).ToList();
    }
    
    // 获取构件详情
    public List<ComponentInfo> GetComponents(string buildingId)
    {
        return _atomicComponents.Where(c => c.source_building == buildingId).ToList();
    }
    
    // 获取风格材料
    public MaterialPalette GetStyleMaterials(string style)
    {
        return _styleMaterials.FirstOrDefault(m => m.style == style);
    }
}

private ToolResult SearchBuildings(string style, string buildingType, string size, KnowledgeBaseManager kb)
{
    // Step1: SQL精确过滤
    var candidates = kb.Components.SearchBuildings(new BuildingSearchCriteria {
        style = style,
        building_type = buildingType,
        size_category = size
    });

    // Step2: 向量语义排序 (可选)
    if (kb.Components.HasVectors)
    {
        var queryVector = kb.Components.BuildQueryVector(style, buildingType, size);
        candidates = kb.Components.SearchBuildingsByVector(queryVector, 10);
    }

    return new ToolResult { Content = Serialize(candidates) };
}
```

## 5. 向量生成

### 5.1 Python脚本 (Tools/vector/generate_building_embeddings.py)

```python
# 建筑向量生成器
class BuildingVectorizer:
    # 建筑类型关键词
    BUILDING_TYPE_KEYWORDS = {
        "住宅": "house home residential living dwelling",
        "神庙": "temple shrine religious sacred worship",
        "商店": "shop store merchant trading market",
        "塔楼": "tower watchtower lookout vertical tall",
    }
    
    # 风格关键词
    STYLE_KEYWORDS = {
        "中式": "chinese asian oriental dynasty pagoda",
        "日式": "japanese asian oriental sakura cherry",
        "奇幻": "fantasy magical enchanted mystical",
        "中世纪": "medieval castle knight stone brick",
    }
    
    def vectorize(self, text, dim=384):
        """TF-IDF + 关键词增强"""
        vector = [0.0] * dim
        words = self._tokenize(text)
        
        # TF-IDF权重
        for word in words:
            idx = abs(hash(word)) % dim
            weight = self._get_weight(word)
            vector[idx] += weight
        
        # 语义关键词增强
        for key, kw_text in BUILDING_TYPE_KEYWORDS.items():
            if key in text:
                for kw in kw_text.split():
                    idx = abs(hash(kw)) % dim
                    vector[idx] += 2.5
        
        # 归一化
        mag = math.sqrt(sum(v * v for v in vector))
        if mag > 0:
            vector = [v / mag for v in vector]
        
        return vector
```

### 5.3 向量生成示例

```python
def generate_building_embedding(building):
    """为单个建筑生成向量"""
    # 组合多个字段作为向量文本
    text_parts = [
        building['name'],
        building['building_type'],
        building['style'],
        building['size_category'],
        building['summary'],
        f"尺寸{building['width']}x{building['height']}"
    ]
    
    # 添加构件信息
    for component in building['components'][:10]:
        text_parts.append(component['type'])
        if component['subtype']:
            text_parts.append(component['subtype'])
    
    text = ' '.join([p for p in text_parts if p])
    
    # 使用TF-IDF + 关键词增强
    embedding = vectorizer.vectorize(text)
    return embedding
```

## 8. 性能优化

### 8.1 内存缓存

向量数据加载到内存字典，避免每次查询读取文件：

```csharp
private Dictionary<string, float[]> _buildingVectors;  // building_id → vector
private List<BuildingInfo> _buildingIndex;              // 建筑索引缓存
private List<ComponentInfo> _atomicComponents;          // 构件缓存
```

### 8.2 混合检索优势

| 纯SQL | 纯向量 | 混合检索 |
|-------|--------|----------|
| 精确但无语义 | 语义但慢 | 精确+语义 |
| O(n)扫描 | O(n)相似度计算 | SQL缩小候选 + 向量排序Top-K |

### 8.3 数据量优化

当前数据库经过筛选，只保留建筑相关数据：
- buildings: 28个完整建筑模板
- atomic_components: 125个可复用构件
- vectors: 28个建筑向量 (384维)
- 减少向量计算量，提升检索速度

## 10. 故障排查

| 问题 | 解决方案 |
|------|----------|
| 向量文件不存在 | 运行 `python Tools/vector/generate_building_embeddings.py` |
| 向量检索返回空 | 检查向量是否正确加载，降低相似度阈值(默认0.3) |
| SQL查询慢 | 添加索引 `CREATE INDEX idx_buildings_style ON buildings(style)` |
| 风格匹配不准 | 补充关键词，检查 building_embeddings.json 是否最新 |
| 建筑数据太少 | 运行 `python Tools/database/manage_buildings_v3.py` 添加模板 |
